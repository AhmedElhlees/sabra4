# تشغيل تطبيق Ibn-alSabrah محلياً (لتفعيل التشفير الكامل والإشعارات الحقيقية)

فتح الملف مباشرة بالنقر المزدوج (`file://`) **يعطّل**: تسجيل الـ Service Worker، الإشعارات الحقيقية، وبعض ميزات التشفير. لتفعيلها كلها لازم تشغّل التطبيق عبر سيرفر محلي بسيط على `http://localhost`.

## الخطوة 1: رتّب الملفات في مجلد واحد
حط الملفات الثلاثة التالية **بنفس المجلد**:
```
malia-app/
├── index.html   ← أعد تسمية ملف التطبيق (malia-app__xx_.html) إلى index.html بالضبط
├── sw.js
└── README.md
```
⚠️ التسمية `index.html` **إلزامية** — الـ manifest المضمّن بالتطبيق مضبوط على `start_url: ./index.html`، فلو أبقيت الاسم الأصلي (`malia-app__12_.html` مثلاً) لن يعمل التثبيت كـ PWA بشكل صحيح.

## الخطوة 2: شغّل سيرفر محلي (اختر طريقة واحدة)

**لو عندك Python مثبّت** (الأغلب موجود افتراضياً على أغلب الأجهزة):
```bash
cd malia-app
python3 -m http.server 8080
```

**لو عندك Node.js مثبّت:**
```bash
cd malia-app
npx http-server -p 8080
```

## الخطوة 3: افتح التطبيق من المتصفح
اذهب إلى:
```
http://localhost:8080
```
(مو `file://` ومو رابط مباشر للملف — لازم `http://localhost` بالضبط حتى يشتغل الـ Service Worker والتشفير الكامل).

## الخطوة 4 (اختياري): ثبّت التطبيق كـ PWA على جوالك
من نفس شبكة الواي فاي، افتح على جهاز الكمبيوتر عنوان IP المحلي بدل `localhost` (مثلاً `http://192.168.1.5:8080`) من متصفح الجوال، ثم اختر "إضافة إلى الشاشة الرئيسية". بهيك التطبيق يفتح متل تطبيق مستقل ويقدر يعرض إشعارات حتى لو المتصفح مو مفتوح فعلياً (بحدود ما يدعمه نظام جوالك).

## ملاحظة مهمة عن حدود الإشعارات
هذا التطبيق **لا يستخدم خادم Push خارجي** — كل الإشعارات (أدوية، ديون، اشتراكات، مهام) تُطلق من صفحة التطبيق نفسها عبر فحص دوري كل دقيقة. هذا يعني:
- ✅ إذا كان التطبيق مفتوحاً بتبويب (حتى بالخلفية) أو مثبّتاً كـ PWA يعمل، ستصلك الإشعارات فعلياً كإشعارات نظام حقيقية.
- ❌ إذا أغلقت المتصفح تماماً (كل التبويبات)، لن تصلك إشعارات جديدة حتى تفتحه مرة أخرى — لأن لا يوجد سيرفر خارجي يوقظ التطبيق. هذا قيد طبيعي لأي تطبيق محلي بدون سيرفر backend حقيقي، وليس خللاً بالتطبيق.

## استكشاف الأخطاء
- إذا ظهرت رسالة فشل تسجيل Service Worker بالـ Console: تأكد إنك تفتح الرابط عبر `http://localhost` بالضبط وليس عبر `file://` أو عبر IP بدون https في حال كان جهازك يفرض ذلك (localhost معفى من شرط https).
- إذا لم تصلك أي إشعار: تأكد أنك سمحت بإذن الإشعارات عندما طلبه التطبيق أول مرة (تحقق من إعدادات المتصفح → أذونات الموقع).
